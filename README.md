Validador de Códigos QR Seguros 

Este proyecto consiste en una aplicación web de ciberseguridad diseñada para auditar, analizar y evaluar el nivel de riesgo de los enlaces (URLs) contenidos dentro de códigos QR antes de que el usuario acceda a ellos. El objetivo principal es mitigar los ataques de **Qshing** (Phishing a través de códigos QR), protegiendo la integridad de los usuarios mediante un motor de análisis heurístico en tiempo real.

El sistema opera bajo un flujo de tres capas de seguridad cuando se procesa una imagen: primero, utiliza la librería `jsQR` para la **decodificación gráfica** de la matriz de puntos y la extracción de la cadena de texto plano; segundo, ejecuta un bloque de **sanitización y validación** mediante expresiones regulares para asegurar que cumpla con la estructura oficial de una URL (`Protocolo + Dominio + Ruta`), bloqueando de inmediato cualquier formato inválido; y tercero, procesa la URL a través de un **motor heurístico de puntuación** basado en reglas estadísticas que asigna puntos de penalización según las anomalías detectadas.

Dentro de esta matriz de evaluación, el nivel de peligro se calcula dinámicamente acumulando puntos de riesgo: la **detección de acortadores** (`bit.ly`, `tinyurl.com`) suma **+2.0 puntos** por ocultamiento de la URL real; los **TLDs de alto riesgo** (extensiones de dominio baratas y masivamente usadas para fraudes como `.online`, `.xyz` o `.top`) suman **+2.0 puntos**; las **palabras de riesgo** asociadas a apuestas o estafas (`casino`, `bonus`, `bet`) añaden **+1.0 punto**; y la **saturación de parámetros** (exceso de variables de seguimiento tras el delimitador `&`) penaliza con **+1.5 puntos**. Dependiendo de la sumatoria, el sistema emite un veredicto visual inmediato en la interfaz: **Seguro (Verde)** si es menor a 1.0 punto, **Sospechoso (Amarillo)** si se encuentra entre 1.0 y 3.0 puntos, y **Malicioso (Rojo)** si supera los 3.0 puntos debido a la acumulación de múltiples amenazas críticas.

Durante las fases de prueba e integración, el software superó importantes **desafíos de ingeniería** para optimizar su lógica de red y reducir los falsos positivos. El primer desafío fue la **mitigación de falsos positivos**, ya que el algoritmo inicial penalizaba de forma rígida cualquier "dominio desconocido", marcando por error los códigos QR de restaurantes o negocios locales nuevos como maliciosos; esto se solucionó eliminando las listas blancas restrictivas y migrando hacia un análisis puramente estructural y estadístico de los TLDs peligrosos. El segundo desafío fue la **optimización del filtro de longitud**, dado que el script original sumaba riesgo a cualquier URL de más de 100 caracteres, lo que rompía los enlaces legítimos de *Google Maps* o menús digitales extensos; la solución de ingeniería consistió en rediseñar el filtro para contar de forma inteligente caracteres delimitadores (`&`) en lugar de las letras totales, logrando medir el rastreo agresivo u ocultamiento de variables sin perjudicar el tamaño del enlace.

Finalmente, como capa de seguridad avanzada para evitar que la propia aplicación sea utilizada como un vector de ataque, se implementó la **mitigación de ataques de Inyección de Código (XSS)** mediante la técnica de *escapar HTML*. Si un atacante intenta inyectar un script malicioso dentro del código QR (por ejemplo, `<script>robarDatos()</script>`), la función intercepta el texto y neutraliza los caracteres especiales de programación transformando los signos `<` y `>` en las entidades seguras `&lt;` y `&gt;`. Esto garantiza que el navegador web procese el contenido estrictamente como texto plano inofensivo que se dibuja en pantalla sin ejecutarse, anulando por completo la vulnerabilidad y logrando un balance perfecto entre una protección robusta y una excelente usabilidad diaria para el usuario.
